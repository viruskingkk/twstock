# 台股六脈神劍選股系統

目前版本：**產業名稱修正 + 離線股本週轉率版**

這是一套本機執行的台股技術面選股工具，整合「六脈神劍、求敗九劍、十八漲」、多空勝率、產業分類、持股追蹤與賣出/贖回提示。  
工具可直接用 HTML 開啟，也可先用 Python 下載 `history.csv` 後離線分析，避免瀏覽器 CORS 與官方 API 限流問題。

> 本工具僅作為技術指標篩選與持股追蹤輔助，不構成投資建議。

---

## 2. 快速開始

### 方式一：直接使用 HTML 預設股票池

適合測試台灣50、少量股票或手動代號清單。

1. 解壓縮 ZIP。
2. 用 Chrome 或 Edge 開啟 `台股六脈神劍選股系統.html`。
3. 在「① 股票池與資料來源」確認股票代號。
4. 按「開始抓取並選股」。
5. 等待分析完成後查看：
   - 六脈神劍
   - 求敗九劍
   - 十八漲
   - 多空勝率
   - 持股追蹤

> 若瀏覽器直接抓官方資料失敗，通常是 CORS 或限流，請改用方式二的 CSV 離線分析。

---

### 方式二：Python 下載 CSV 後離線分析，推薦全市場使用

1. 安裝 Python 3.10 以上。
2. 雙擊：

```text
download_common_stocks_with_capital.bat
```

或手動執行：

```bash
python tw_stock_downloader.py --all --out history.csv --months 4 --sleep 0.8 --resume
```

3. 下載完成後會產生：

```text
history.csv
stock_meta.csv
```

4. 開啟 `台股六脈神劍選股系統.html`。
5. 找到「上傳離線下載的 CSV 資料」。
6. 選擇 `history.csv`。
7. 按「用此 CSV 選股」。

## 3. history.csv 與 stock_meta.csv 要怎麼用？

### history.csv

這是 HTML 主要要匯入的檔案。新版欄位如下：

```csv
code,name,market,industry,capital,date,open,high,low,close,volume
2330,台積電,twse,24 半導體業,2593.04,2026-06-28,1000,1010,995,1005,25000000
```

HTML 匯入 `history.csv` 後會自動完成：

- 產業分類
- 股本讀取
- 週轉率計算
- 技術指標計算
- 多空勝率計算
- 持股追蹤比對

### stock_meta.csv

這是備查檔，不需要放到 HTML，也不用上傳。

用途是讓你用 Excel 檢查股票基本資料：

```csv
code,name,market,industry,capital
2330,台積電,twse,24 半導體業,2593.04
```

如果你只想分析，**只要上傳 `history.csv` 即可**。

## 4. 週轉率計算邏輯

週轉率不是官方日行情直接提供，而是用成交量與股本離線計算。

公式：

```text
週轉率 = 成交量(張) ÷ 股本(億) ÷ 100
```

新版 downloader 會把 `capital` 寫進 `history.csv`，所以 HTML 不需要再即時抓股本，也比較不會出現整排 `⚠️無股本`。

如果仍顯示 `⚠️無股本`，通常代表該股票在 CSV 裡的 `capital` 欄位是空值。

## 5. 產業分類

新版已修正產業分類顯示，會把代碼轉成中文名稱。

範例：

```text

25 -> 25 金融保險業
23 -> 23 航運業
```

因此畫面不會只顯示 `01`、`02`、`14` 這類看不懂的代碼。

產業分類功能支援：

- 產業搜尋
- 全選
- 全不選
- 套用篩選
- 依產業縮小股票池

## 6. 股票池與名冊篩選

新版 downloader 的 `--all` 預設只抓：

- 上市普通股
- 上櫃普通股

預設排除：

- ETF
- ETN
- 權證
- 債券型商品
- 受益證券
- 其他非普通股商品

正常普通股全市場大約會落在兩千檔上下，不應該接近一萬檔。  
如果看到 10000 檔左右，通常是跑到舊版 downloader 或使用了過寬名冊參數。

### 包含 ETF

如需包含 ETF，可執行：

```bash
python tw_stock_downloader.py --all --include-etf --out history.csv --months 4 --sleep 0.8 --resume
```

## 7. 選股規則說明

### 六脈神劍

六脈神劍分成多方與空方。

多方：

```text
第一劍【翻多】
第三劍【轉強】
第五劍【收高】
```

空方：

```text
第二劍【翻空】
第四劍【轉弱】
第六劍【收低】
```

每個規則區塊支援：

- 收合 / 展開
- 篩選本條件
- 全選 / 全不選
- 個股加入追蹤
- 外部網站連結

### 求敗九劍

求敗九劍主要視為空方或轉弱條件，用於辨識風險、減碼與停損觀察。

### 十八漲

十八漲前 12 條視為多方條件。

第 13～18 條屬於排行榜型規則，系統將其視為參考訊號，不直接灌入主共振分數，避免小股票池時每檔股票都被選中。

## 8. 多空勝率與分類篩選

「多空勝率」是根據目前技術規則命中數推估的方向機率，不是歷史回測保證勝率。

畫面會顯示：

- 分析股票數
- 偏多以上
- 偏空以上
- 多空打架
- 強多
- 偏多
- 中性觀察
- 偏空
- 強空

統計卡片可以點擊篩選，支援分類呈現。

分類意義：

| 分類 | 說明 |
|---|---|
| 強多 | 多方條件明顯大於空方條件 |
| 偏多 | 多方條件較多，但仍需觀察 |
| 中性觀察 | 多空訊號不明顯 |
| 偏空 | 空方條件較多，需注意風險 |
| 強空 | 空方條件明顯，偏向避開、減碼或賣出觀察 |
| 多空打架 | 多方與空方同時明顯，波動或轉折風險較高 |

## 9. 持股追蹤

可以把已買進股票輸入到持股追蹤區。

格式：

```text
代號,買進價,股數,購買金額,備註
```

範例：

```text
3481,18.5,1000,18500,面板追蹤
2330,950,,95000,長線核心
```

若只填購買金額、不填股數，系統會自動估算：

```text
股數 = 購買金額 / 買進價
```

持股追蹤會顯示：

- 現價
- 買進價
- 股數
- 購買金額
- 目前市值
- 損益金額
- 損益百分比
- 多方機率
- 空方機率
- 賣出 / 贖回提示

## 10. 賣出 / 贖回提示

可設定：

- 空方機率達幾 % 時提醒賣出 / 贖回
- 停損跌幅幾 % 時提醒停損 / 贖回

提示類型包含：

```text
續抱
續抱觀察
減碼觀察
賣出 / 贖回
停損 / 贖回
```

## 12. 常見問題

### Q1：stock_meta.csv 要放在哪？

不用放，也不用上傳。  
HTML 只需要匯入 `history.csv`。  
`stock_meta.csv` 是備查檔，方便你檢查產業、名稱、股本是否正確。

### Q2：為什麼產業只顯示 01、02、14？

那是舊版直接顯示產業代碼。新版會自動轉成中文產業名稱，例如 `14 半導體業`。

### Q3：為什麼直接用 HTML 抓資料會失敗？

瀏覽器有 CORS 限制，TWSE / TPEx 部分 API 不允許本機 HTML 直接抓取。  
全市場建議用 Python downloader 先產生 `history.csv`，再匯入 HTML。

### Q4：為什麼以前抓到 10989 檔？

舊版名冊抓太寬，把 ETF、ETN、權證、債券、受益證券等非普通股商品一起抓進來。新版 `--all` 已預設只抓普通股。

### Q5：CSV 選了但讀不到怎麼辦？

請改用備援方式：

1. 用記事本打開 `history.csv`。
2. 全選複製。
3. 貼到 HTML 裡的 CSV 備援文字區。
4. 按「用此 CSV 選股」。

### Q6：多空勝率可以當成真正勝率嗎？

不建議。這是根據規則命中數推估的方向機率，適合排序、追蹤風險與提醒觀察，不是歷史回測保證勝率。

## 14. 注意事項

- 本工具不是投資建議。
- 技術指標需要足夠交易日資料，建議至少 30 個交易日以上。
- 全市場資料量大，建議使用 CSV 離線模式。
- 官方 API 可能限流，下載時建議保留 `--sleep 0.8` 或更高。
- 台灣50成分股與上市櫃名冊會定期變動，請定期重新下載資料。
